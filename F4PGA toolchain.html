F4PGA toolchain

Introduction

F4PGA is a fully opensources of an end-to-end FPGA sythesis toolchain, it provides all necessary tools to convert input HDL into the final bitstream (bitstream format is depends on the platform that you used).

Follow this tutorial, you just need to do copy-paste in terminal to finish it, sounds easy right? :)

Contents

  1. Install F4PGA and all of its dependencies,
  2. Build and upload example designs onto the devboard of your choice.
  3. Compile and run your own designs using the F4PGA toolchain.
  4. Customize the Makefile for your own designs.

Prerequisite

  1. Ubuntu-desktop 18.04 / 20.04.4 (We demo on this version)
  2. Python v3.8
  3. FPGA board (upload your bitstream to test the result)

Installation steps

  1. Clone f4pga-examples reposity.
  2. Get Miniconda package manager.
  3. Downloading F4PGA toolchain.
  4. Add the architecture definition.
  5. Get to working on conda environment.
  6. Building another architecture definition.
  7. Make bitstream with led example.
  8. Load bitstream in your board to realized your function

1. Clone f4pga-examples reposity

Before you starting, please update and install the ubuntu tool:
(-y means you choose “yes” of this command)

sudo apt update -y 
sudo apt install -y git wget xz-utils

Then you have tool to get the f4pga-examples from there github:

git clone https://github.com/chipsalliance/f4pga-examples

2. Get Miniconda package manager

Download conda installer script( .sh file) into ~/f4pga-examples:

cd f4pga-examples
wget https://repo.continuum.io/miniconda/Miniconda3-latest-Linux-x86_64.sh -O conda_installer.sh

Choose install directory, you can either install in home ~/opt/f4pga or
in system ( you need to add sudo in head of your command, but it’s really troublesome )/opt/f4pga, here we install in home directory:

export INSTALL_DIR=~/opt/f4pga

Select target your FPGA family(“xc7” means Xilinx 7-Series):

export FPGA_FAM=xc7

Setup conda and your environments:

bash conda_installer.sh -u -b -p $INSTALL_DIR/$FPGA_FAM/conda;
source "$INSTALL_DIR/$FPGA_FAM/conda/etc/profile.d/conda.sh";
conda env create -f $FPGA_FAM/environment.yml

3. Download F4PGA toolchain

Download F4PGA package:

mkdir -p $INSTALL_DIR/xc7/install
wget -qO- https://storage.googleapis.com/symbiflow-arch-defs/artifacts/prod/foss-fpga-tools/symbiflow-arch-defs/continuous/install/20220404-212755/symbiflow-arch-defs-install-afbfe04.tar.xz | tar -xJC $INSTALL_DIR/xc7/install

4. Add the architecture definition

Now, add your archietecture, you can choose different board follow as GCS bucket, we used xc7z020_test as an example, and you’ll get target xc7z020clg400-1 arch-defs

mkdir -p $INSTALL_DIR/xc7/install
wget -qO- https://storage.googleapis.com/symbiflow-arch-defs/artifacts/prod/foss-fpga-tools/symbiflow-arch-defs/continuous/install/20220404-212755/symbiflow-arch-defs-xc7z020_test-afbfe04.tar.xz | tar -xJC $INSTALL_DIR/xc7/install

5. Get to working on conda environment

Before executing instructions of F4PGA, setting the installation directory to match what you set it to earlier, assigning the path and source for your conda environment, and activating your environment:

export INSTALL_DIR=~/opt/f4pga
export FPGA_FAM=xc7
source "INSTALL_DIR/$FPGA_FAM/conda/etc/profile.d/conda.sh"
conda activate $FPGA_FAM

6. Building another architecture definition

We’ll get a pinmap.csv of this step, so if you already have raw data of it, you can copy it into the corresponding folder and skip this step.

We would like to get some others target of arch-defs, so we git Symbiflow resource to build it:

git clone https://github.com/f4pga/f4pga-arch-defs.git https://github.com/f4pga/prjxray-db.git
cd symbiflow-arch-defs
make env

Next, we make an example of zedboard(xc7z020clg484-1) architecture:

cd build/xilinx/xc7/archs/zynq7_z020

7. Make bitstream with led example

To demo led example, you need to prepared following three files:

  1. led.v
  2. led.xdc
  3. Makefile

You can make a folder in ~/f4pga-examples first

mkdir -p ~/f4pga-examples/led
cd ~/f4pga-examples/led

Text nano led.v to create verilog file, and copy the code:
(Demo with dataflow model)

module top(
    output reg[7:0] led,
    input [7:0] switch,
    input clk);
    
  always @(posedge clk) begin
      led = switch;
  end
endmodule

Text nano led.xdc to create xdc file, and copy the code:

set_property IOSTANDARD LVCMOS33 [get_ports led]
set_property IOSTANDARD LVCMOS25 [get_ports switch]
set_property IOSTANDARD LVCMOS33 [get_ports clk]

set_property PACKAGE_PIN T22 [get_ports {led[0]}]
set_property PACKAGE_PIN T21 [get_ports {led[1]}]
set_property PACKAGE_PIN U22 [get_ports {led[2]}]
set_property PACKAGE_PIN U21 [get_ports {led[3]}]
set_property PACKAGE_PIN V22 [get_ports {led[4]}]
set_property PACKAGE_PIN W22 [get_ports {led[5]}]
set_property PACKAGE_PIN U19 [get_ports {led[6]}]
set_property PACKAGE_PIN U14 [get_ports {led[7]}]

set_property PACKAGE_PIN F22 [get_ports {switch[0]}]
set_property PACKAGE_PIN G22 [get_ports {switch[1]}]
set_property PACKAGE_PIN H22 [get_ports {switch[2]}]
set_property PACKAGE_PIN F21 [get_ports {switch[3]}]
set_property PACKAGE_PIN H19 [get_ports {switch[4]}]
set_property PACKAGE_PIN H18 [get_ports {switch[5]}]
set_property PACKAGE_PIN H17 [get_ports {switch[6]}]
set_property PACKAGE_PIN M15 [get_ports {switch[7]}]

set_property PACKAGE_PIN Y9 [get_ports {clk}]
create_clk -period 10 [get_ports clk]

Text nano Makefile to create makefile, and copy the code:

current_dir := ${CURDIR}
TARGET := zedboard
TOP := top
SOURCES := ${current_dir}/led.v

XDC := ${current_dir}/led.xdc

include ~/f4pga-examples/common/common.mk

Before you make bitstream, you should check something:

  1. You already in the conda environment. If not, goto step 5.
  2. Copy the architectures folder into /usr/local/:
sudo mkdir -p /usr/local/xc7/share/
cp -r ~/opt/f4pga/xc7/install/share/symbiflow/ /usr/local/xc7/share
mv /usr/local/xc7/share/symbiflow /usr/local/share/f4pga
  1. Typesudo nano ~/f4pga-examples/common/common.mk to add your board architecture information into if-else space and save it:
else ifeq($(TARGET),zedboard)
    DEVICE := xc7z020_test
    BITSTREAM_DEVICE := zynq7
    PARTNAME := xc7z020clg484-1

Now, make bitstream. If it sucessful, you’ll see /build/zedboard/ has a file named top.bit:

cd ~/f4pga-examples/led/ make

8. Load bitstream in your board to realized your function

To load your bitstream in board, you need to install a module named openocd, here we suggest to install manual:

cd ~
git clone https://github.com/openocd-org/openocd.git
sudo apt-get install libtool libusb-1.0-0-dev make
cd openocd ./boostrap ./configure --enable-ftdi
make
sudo make install

Make sure your device is connected, and change mode of it:

lsusb
dmesg | grep tty*
sudo chmod 666 /dev/ttyUSB0

After finish it, load bitstream that you created into zedboard:

cd ~/f4pga-examples/led/build/zedboard
openocd -f ~/openocd/tcl/board/digilent_zedboard.cfg -c "init; pld load 0 top.bit; exit"
